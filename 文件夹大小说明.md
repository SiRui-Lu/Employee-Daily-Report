# 项目文件夹大小说明

## 📊 各文件夹大小

根据检查结果：

| 文件夹 | 大小 | 说明 |
|--------|------|------|
| **node_modules** | ~746 MB (0.75 GB) | 所有 npm 依赖包 |
| **release** | ~482 MB (0.48 GB) | Electron 打包后的文件 |
| **dist** | ~1 MB | 构建后的前端文件 |
| **其他文件** | < 1 MB | 源代码和配置文件 |

**总计：约 1.2 GB**

## 🔍 详细说明

### 1. **node_modules** (~746 MB) - 开发依赖
- **作用**：包含所有 npm 安装的依赖包
- **是否可以删除**：可以删除，但需要时运行 `npm install` 重新安装
- **建议**：
  - ✅ 开发时需要保留
  - ✅ 已添加到 `.gitignore`，不会提交到 Git
  - ❌ 分发项目时不需要（用户只需要安装后的 exe）

### 2. **release** (~482 MB) - 打包输出
- **作用**：包含 Electron 打包后的安装程序和可执行文件
- **是否可以删除**：可以删除，需要时运行 `npm run dist` 重新生成
- **建议**：
  - ✅ 已添加到 `.gitignore`，不会提交到 Git
  - ✅ 只需要保留最终的 `.exe` 安装程序文件
  - ❌ `win-unpacked` 文件夹可以删除（如果需要便携版可以保留）

### 3. **dist** (~1 MB) - 构建输出
- **作用**：Vite 构建后的前端文件
- **是否可以删除**：可以删除，需要时运行 `npm run build` 重新生成
- **建议**：
  - ✅ 已添加到 `.gitignore`
  - ✅ Electron 打包时需要

## 💡 如何减小项目大小

### 方案一：删除不需要的文件夹（推荐）

如果只是开发，可以删除：
```bash
# 删除打包输出（可以重新生成）
rmdir /s /q release

# 注意：不要删除 node_modules，否则需要重新 npm install
```

### 方案二：分发时只保留必要文件

如果要分发给其他开发者，只需要：
- ✅ 源代码文件（`.tsx`, `.ts`, `.json` 等）
- ✅ `package.json` 和 `package-lock.json`
- ❌ 不需要 `node_modules`（让用户自己运行 `npm install`）
- ❌ 不需要 `dist` 和 `release`（可以重新生成）

### 方案三：使用 .gitignore（已配置）

这些大文件夹已经在 `.gitignore` 中：
- `node_modules/`
- `dist/`
- `release/`

所以提交到 Git 时不会包含这些文件夹。

## 📦 最终打包文件大小

用户安装后的应用大小：
- **安装程序**：`校园洗涤员工日报系统 Setup 1.0.0.exe` (~100-150 MB)
- **安装后**：应用本身约 200-300 MB（包含 Electron 运行时）

## ✅ 总结

1. **1GB 大小是正常的**，主要是：
   - `node_modules` (746 MB) - 开发依赖
   - `release` (482 MB) - 打包输出

2. **可以安全删除**：
   - `release/` 文件夹（可以重新打包）
   - `dist/` 文件夹（可以重新构建）

3. **不要删除**：
   - `node_modules/`（除非确定不再开发）

4. **分发时**：
   - 只需要源代码 + `package.json`
   - 用户运行 `npm install` 安装依赖
   - 用户运行 `npm run dist` 打包
